Linux

网络连接的三种方式

桥接模式

虚拟系统可以和外部系统通讯，但是容易造成ip冲突(超过255)

NAT模式

NAT是网络地址转换，相当于给你虚拟机虚拟一个ip，然后在需要访问外部的时候，自动转换成你母机ip去访问外部，但是外部不可以直接访问内部

主机模式

相当于一个独立的系统，不与外界发生联系

Linux目录结构

  • /bin (/usr/bin、/usr/local/bin),是Binary的缩写，存放着最常使用的命令
  • /sbin 存放的是系统管理员使用的系统管理程序
  • /home 存放普通用户的主目录，在linux中每个用户都有自己的目录
  • /root 系统管理员，也成为超级权限者的用户主目录
  • /lib 系统开机所需要的最基本的动态连接共享库
  • /lost+found 一般情况是空的，当系统非法关机后，这里就存放了一些文件
  • /etc 所有的系统管理所需要的配置文件和子目录，例如mysql的my.conf
  • /usr 用户的很多应用程序和文件都放在这个目录下，类似于windows的program files目录
  • /boot 存放的是启动linux时使用的一些核心文件，包括一些连接文件以及镜像文件
  • /proc 是一个虚拟目录，是系统内存的映射，访问这个目录来获取系统信息
  • /srv service的缩写，存放一些服务启动之后需要提取的数据
  • /sys linux2.6之后出现，安装了一个文件系统sysfs
  • /tmp 用来存放一些临时文件
  • /dev 类似于Windows的设备管理器，把所有的硬件用文件形式存储
  • /media 会自动识别一些例如u盘光驱等设备，识别后就将识别的设备挂载到这目录下
  • /mnt 该目录是为了让用户挂载别的文件系统，可以将外部存储挂载在/mnt上，进入该目录查看
  • /opt 给主机额外安装软件的目录
  • /usr/local 另一个给主机额外安装软件所安装的目录，一般是通过编译源码方式安装的程序
  • /var 存放一些不断扩张的东西，习惯将经常被修改的目录放在这个目录下，包括各种日志文件
  • /selinux[security-enhanced linux] 是一种安全的子系统，它能控制程序只访问特定文件，有三种工作模式，可以自行设置

vi和vim编辑器

常用的三种模式

正常模式

是vim的默认模式，这个模式中可以使用上下左右来移动光标，可以使用删除或删除整行来处理内容，也可以复制粘贴

插入模式

按下i、I、o、O、a、A、r、R中任意一个字母进入编辑模式，一般按i即可

命令行模式

输入esc退出当前模式，再输入:进入命令行模式

可以提供相关指令，完成读取，存盘，替换，离开vim显示行号等动作。

在一般模式下:

  • dd 删除当前行；5dd，删除当前向下的5行
  • G 文档最末行，gg 最首行
  • 先输入行号 在输入shift+g 表示移动到多少行

在命令行模式下:

开机重启与用户登录注销

开关机与重启

  • shutdown -h now 立刻关机
  • shutdown -h 1 1分钟后关机
  • shutdown -r now 立即重启
  • halt 现在关机
  • reboot 现在重启
  • sync 把内存的数据同步到磁盘

不管重启还是关机，都会先进行sync命令，目前的大部分命令都会在关机前执行sync，但是再执行一次较好

直接输入shutdown默认为shutdown -h 1

linux用户管理

linux系统是一个多用户多任务的操作系统，任何一个要使用系统资源的用户，都必须首先向系统管理员申请一个账号，然后以这个账号的身份进入系统

用户基本操作

添加用户
  • useradd 用户名 添加用户
  • useradd -g 用户组 用户名 增加用户时直接加上组

创建成功后，会自动创建和用户名相同的home目录，也可以通过useradd -d 指定目录，新的用户名，给新创建的目录指定home目录

如果直接添加用户而不指定组，则虚拟机会默认创建一个以用户名为名的组并将其放入组中

修改/指定密码

passwd 用户名 给某个用户设置密码

pwd命令是显示当前用户所在目录

删除用户
  • userdel 用户名 删除用户但保留对应home目录
  • userdel -r 用户名 删除用户并且删除对应home目录
查询用户信息
  • id 用户名
切换用户

  • su - 切换的用户名 切换用户，高权限切低权限不需要密码，反之需要

  • exit/logout 返回到原来用户

查看当前用户
  • whoami/who am i 查看当前用户，不加空格是当前操作的用户，加空格是第一次登录用户

用户组

简介

类似于角色，系统可以对有共性(权限)的多个用户进行统一的管理

新增组
  • groupadd 组名 新增组
  • useradd -g 用户组 用户名 增加用户时直接加上组

如果直接添加用户而不指定组，则虚拟机会默认创建一个以用户名为名的组并将其放入组中

修改用户的组
  • usermod -g 用户组 用户名

用户和组相关文件

  • /etc/passwd 文件

用户(user)的配置文件，记录用户的各种信息

每一行都是一个用户：含义: 用户名:口令:用户标识号:组标识号:注释性描述:主目录:登录shell

  • /etc/shadow文件

口令配置文件,每行的含义:登录名:加密口令:最后一次修改时间:最小时间间隔:最大时间间隔:警告时间:不活动时间:失效时间:标志

  • /etc/group文件

组的配置文件，记录linux中组的信息，每行含义:组名:口令:组标识号:组内用户列表

指令

运行级别

运行级别说明

  1. 关机

  2. 单用户[找回丢失的密码]

  3. 多用户状态没有网络服务

  4. 多用户状态有网络服务

  5. 系统未使用保留给用户

  6. 图形界面

  7. 系统重启

常用的运行级别是3和5

  • init[0123456] 切换运行级别
  • systemctl get-default 查看目前的运行级别
  • systemctl set-default mult-user.target/systemctl set-default graphical.target 设置允许级别，前者相当于3，后者相当于5

找回root密码

  1. 启动系统，进入开机页面，在页面中按‘e’进入编辑界面

  2. 进入之后使用键盘的上下键移动光标，找到以“Linux 16”开头所在的行数，在行的最后输入:init=/bin/sh

  3. 输入之后按快捷键Ctrl + x 进入单用户模式

  4. 在光标闪烁的位置输入mount -o remount,rw / (单词间有空格)，然后按回车

  5. 在新的一行最后面输入passwd。完成后按回车输入密码，然后再次确认密码即可

  6. 接着在鼠标闪烁的位置输入: touch /.autorelabel(touch 后面又空格),之后继续按回车

  7. 接着在光标闪烁的位置输入 exec /sbin/init 之后按回车键，等待系统修改密码，完成后系统会自动重启

组管理和权限管理

在linux中每个用户必须属于一个组，不能独立于组外，在linux中每个文件有所有者，所在组，其他组的概念

文件/目录所有者

一般为文件的创建者，谁创建了该文件，就成为该文件的所有者

  • ls -ahl 查看文件的所有者
  • chown 用户名 文件名 修改文件所有者
  • chown 用户名: 组名 修改文件/目录的所有者和所在组 -R递归

创建组

  • groupadd 组名

修改文件所在组

  • chgrp 组名 文件名

改变用户所在组

  • usermod -g 新组名 用户名
  • usermod -d 目录名 用户名 改变该用户登陆的初始目录

rwx权限

例如 -rwxrw-r– 1 root root 1213 Feb 2 09:39 abc

​ rwx分别为可读可写可执行

​ 第零位可能为[d,-,l,c,b] 用来确定文件类型

​ d是目录，相当于文件夹

​ -是普通文件

​ l是链接，相当于快捷方式

​ c是字符设备文件，鼠标，键盘

​ b是块设备，比如硬盘

一到三位确定所有者拥有该文件的权限

四到六位确定所属组拥有的权限

七到九位确定其他用户组拥有的权限

1:文件:硬连接数 目录:子目录数(包括隐藏的.和..)+子文件数

root: 用户

root: 组

12123 文件大小，如果是文件夹，显示4069字节

feb…：最后修改日期

abc:文件名

作用到文件

r:可读，查看

w:可写，但是不代表可以删除该文件，前提条件是对该文件所在目录有写权限，才能删除

x:可执行

作用到目录

r:可读

w:可以修改，对目录内创建+删除+重命名目录

x:可以进入该目录 (意思是可以cd到目录，但不可以ls目录，如果对目录下的文件有修改功能，也是可以 修改的)

权限可用数字表示，r为4，w为2，x为1；

​ u为所有者,g为所在组,o为其他人

​ 有+=-三种方式改变权限

  • chmod u=rwx,g=rx,o=x 文件/目录; == chmod 751 文件/目录

定时任务调度

cornd定时任务

任务调度是指系统在某个时间执行的特定的命令或程序

分类：系统工作:有些重要的工作必须周而复始的执行，如病毒扫描等

​ 个别用户工作:个别用户可能希望执行某些程序，比如mysql程序的备份

  • crontab 进行定时任务的设置 -e编辑定时任务，-l查询定时任务 -r删除当前用户所有的定时任务

占位符:

项目 含义 范围
第一个* 一小时当中的第几分钟 0-59
第二个* 一天当中的第几个小时 0-23
第三个* 一个月当中的第几天 1-31
第四个* 一年当中的第几个月 1-12
第五个* 一周当中的星期几 0-7(0和7都代表星期日)

特殊符号说明

特殊符号 含义
* 代表任何时间，比如第一个*代表一小时中的每一分钟都执行一次的意思
， 代表不连续的时间，比如“0 8,12,16 * * *”代表每天的8:00,12:00,16:00都会执行
- 代表连续的时间范围，如“0 5 * * 1-6”代表周一到周六的凌晨5:00执行命令
*/n 代表每隔多久执行一次,如“*/10 * * * *”代表每隔十分钟执行一次命令

磁盘分区与挂载

linux只有一个根目录，一个独立且唯一文件结构，Linux中每个分区都是用来组成整个文件系统的一部分

linux采用了一种叫”载入”的处理方法，他的整个文件系统中包含了一整套的文件和目录，且将一个分区和一个目录联系起来，这时要载入的一个分区将使得它的存储空间在一个目录下获得

  • lsblk 查看所有设备挂载情况 -f可以看更详细的内容
  • fdisk 分区，m显示命令列表；p显示磁盘分区，同fdisk -l；n新增分区，d删除分区，w写入并推出； 过程：开始分区后输入n新增分区，然后选择p，分区类型为主分区，两次回车默认剩余全部空间，最后输入w写入分区并退出，若不报存退出输入q
  • mkfs 格式化分区 mkfs [选项] [-t 分区格式] 分区路径

  4. 如果均没有，则到域名服务DNS进行解析域

进程管理

概述

  • 在Linux中，每个执行的程序都称为一个进程，每个进程都分配一个ID号(pid，进程号)
  • 每个进程都可能以两种方式存在的。前台和后台，所谓前台进程就是用户目前的屏幕上可以进行操作的。后台进程则是实际在操作，屏幕上无法看到，
  • 一般的系统的服务都是以后台进程的方式存在，而且都会常驻在系统中，直到关机才结束

显示系统执行的进程

    • -a显示当前终端的所有进程信息

    • -u以用户的格式显示进程信息

    • -x显示后台进程运行的参数

      一般可以全部组合使用 即: ps -aux 可以配合管道符使用

    • -e显示所有进程

    • -f全格式

      ps -ef是以全格式 显示所有的进程

服务管理

概述

服务的本质就是进程，但是是运行在后台的，通常都会监听某个端口，等待其他程序的请求，比如(mysql sshd 防火墙等),因此我们又称为守护进程。

service管理指令

在CentOS7.0之后，很多服务器都不使用service，而是systemctl

service指令管理的服务在/etc/init.d查看

  • service 服务名 [start|stop|restart|reload|status] 管理服务
  • setup 查看全部服务
  • ls -l /etc/init.d 看到service指令管理的服务
  • chkconfig 可以给服务的各个运行级别设置自启动/自关闭
    • chkconfig –list [|grep xxx] 查看服务
    • chkconfig 服务名 –list
    • chkingconfig –level 5 服务名 on/off
  • systemctl [start|stop|restart|reload|status] 服务名
    • systemctl指令管理的服务在/usr/lib/systemd/system 查看
    • systemctl list-unit-files [|grep 服务名] 查看开机启动状态
    • systemctl enable 服务名 设置服务开机启动
    • systemctl disable 服务名 关闭服务开机启动
    • systemctl is-enabled 服务名 查询某个服务是否是自启动的
    • 这种方式只是临时生效，重启系统后失效，如果想要永久的话，得用enable或disable
服务的运行级别

linux系统有7种运行级别(runlevel)，常用的级别是3和5

运行级别 描述
0级 系统停机状态，系统默认的运行级别不能设为0，否则不能正常启动
1级 单用户工作状态，root权限，用于系统维护，禁止远程登录
2级 多用户状态(没有NFS),不支持网络
3级 完全的多用户状态(有NFS)，登陆后进入控制台命令行模式
4级 系统未使用，保留
5级 x11控制台，登陆后进入图像GUI模式
6级 系统正常关闭并重启，默认运行级别不能设为6，否则不能正常启动

综合实例

要求，在每天凌晨两点半备份数据库文件

#!/bin/bash
#备份目录
BACKUP=/www/xueshuai/db
#获取当前时间
DATETIME=$(date +%Y-%m-%d_%H%M%S)
echo $DATETIME
#文件地址
HOST=localhost
#数据库用户名
DB_USER=root
#数据库密码
DB_PW=123456
#要备份的数据库名
DATABASE=guli
#创建备份目录，如果不存在就创建，存在就使用,${BACKUP}加大括号只是为了和/分开
[ ! -d "${BACKUP}/${DATETIME}" ] && mkdir -p "${BACKUP}/${DATETIME}"
#备份数据库
#                                                                       压缩  覆盖
mysqldump -u${DB_USER} -p${DB_PW} --host=${HOST} -q -R --databases ${DATABASE} | gzip > ${BACKUP}/${DATETIME}/$DATETIME.sql.gz
#将文件处理成tar.gz形式
cd ${BACKUP}
tar -zcvf $DATETIME.tar.gz ${DATETIME}
#删除打包前文件
rm -rf ${BACKUP}/${DATETIME}
#删除十天前的备份文件                   调用其他的命令  \是格式要求
find ${BACKUP} -atime +10 -name "*.tar.gz" -exec rm -rf {} \;
echo "备份数据库${DATABASE} 成功"
